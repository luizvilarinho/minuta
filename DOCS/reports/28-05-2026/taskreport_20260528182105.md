# Task: Última tentativa de correção do fetch de legendas do YouTube

## Objective
Implementar a última tentativa técnica de correção no fluxo atual de YouTube, adicionando instrumentação detalhada da request de legendas e ajustando o contexto de execução para validar se o problema está no mecanismo atual de acesso ao endpoint `timedtext`.

## Context
O app desktop Tauri + React + Rust já possui integração de YouTube em `src-tauri/src/commands/youtube.rs`, baseada em WebView2, cookies de sessão e `eval_with_callback`. Os logs atuais mostram que a watch page é carregada e que `captionTracks` são encontrados, mas as requests para `api/timedtext` retornam body vazio. A análise concluiu que a causa mais provável está no transporte/contexto da request.

## Requirements
- Instrumentar as requests de legenda para capturar, no mínimo, `status`, `statusText`, `responseURL`, headers relevantes e tamanho do body.
- Ajustar o fluxo para testar o fetch das legendas a partir da própria watch page do vídeo, em vez de depender apenas do contexto de `https://www.youtube.com/`.
- Melhorar os logs para diferenciar claramente os cenários de falha da request.
- Manter o restante do fluxo do app intacto.
- Se possível, usar a nova instrumentação para tornar a mensagem de erro mais precisa em casos claros.

## Constraints
- Não alterar o pipeline de resumo nem os outros modos de input.
- Fazer mudanças mínimas e focadas no diagnóstico/correção desta última tentativa.
- Reaproveitar a arquitetura atual baseada em WebView2; não trocar a solução inteira nesta etapa.
- Preservar compatibilidade com Tauri 2 e com o codebase existente.

## Edge Cases
- Request com `status 200` e body vazio.
- Request com redirect silencioso para outra URL.
- Request com `status 0`, `403`, `204` ou resposta bloqueada.
- Signed `baseUrl` falhando, mas fallback simples funcionando.
- Watch page carregando corretamente, mas fetch de legenda falhando por contexto/referer/origem.
