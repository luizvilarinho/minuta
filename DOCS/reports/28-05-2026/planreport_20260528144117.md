## Escopo

Adicionar ao app a capacidade de buscar a transcrição de um vídeo do YouTube (via InnerTube API, usando o crate `yt-transcript-rs` v0.1.8) e alimentá-la no pipeline de resumo já existente — sem modificar `generate_summary` nem nenhum outro comando Rust atual.

O usuário verá um quarto modo de input ("YouTube") no seletor de modos já existente. Ao colar uma URL e confirmar, o app busca a transcrição em Rust e popula o estado `transcription` exatamente como os outros modos fazem — a partir desse ponto o fluxo é idêntico.

### Etapa 2 — Criar o command Rust `fetch_youtube_transcript`

- **Status:** PENDENTE
- **Descrição:** Criar o arquivo `src-tauri/src/commands/youtube.rs` com o conteúdo abaixo. O arquivo deve conter **apenas** o que está descrito — sem abstrações adicionais.

  Responsabilidades do command:

  1. **Extrair o `videoId`** da URL recebida. Suportar os três formatos:
     - `https://www.youtube.com/watch?v=VIDEO_ID` (e variações com outros parâmetros)
     - `https://youtu.be/VIDEO_ID`
     - `https://www.youtube.com/shorts/VIDEO_ID`

     A extração deve ser feita com manipulação de string/URL (sem regex externa). Retornar `Err("URL inválida. Cole um link do YouTube válido.".to_string())` se nenhum formato reconhecido for encontrado ou se o `videoId` extraído tiver comprimento fora de 10–12 caracteres alfanuméricos.

  2. **Buscar as legendas** usando o crate `yt-transcript-rs`. A chamada principal é:
     ```rust
     let fetcher = YtTranscript::new();
     let transcripts = fetcher.get_transcript_list(video_id).await
         .map_err(|e| map_yt_error(e))?;
     ```
     Consultar a API do crate para `get_transcript_list` e como obter a faixa de legenda desejada.

  3. **Selecionar idioma** com fallback: tentar `"pt-BR"`, depois `"pt"`, depois `"en"`, depois a primeira faixa disponível qualquer. Se nenhuma faixa existir, retornar `Err("Este vídeo não possui legendas disponíveis.".to_string())`.

  4. **Concatenar os segmentos** da faixa escolhida em texto puro (apenas o campo de texto de cada segmento, separados por espaço ou `\n`), sem timestamps.

  5. **Função auxiliar `map_yt_error`** que converte os erros do crate em mensagens legíveis ao usuário:
     - Erro de vídeo não encontrado / privado / removido → `"Vídeo não encontrado, privado ou removido."`
     - Erro de PO token requerido → `"Transcrição indisponível para este vídeo (proteção do YouTube)."`
     - Erro HTTP 429 → `"Muitas requisições. Aguarde alguns instantes e tente novamente."`
     - Qualquer outro erro → `format!("Erro ao buscar transcrição: {}", e)`

  A assinatura do command deve ser:
  ```rust
  #[tauri::command]
  pub async fn fetch_youtube_transcript(url: String) -> Result<String, String>
  ```

  Não recebe `AppHandle` nem `State` — não precisa de chave de API.

- **Arquivos:** `src-tauri/src/commands/youtube.rs`
- **Depende de:** Etapa 1

### Etapa 4 — Criar o componente frontend `YoutubeUrlInput.tsx`

- **Status:** PENDENTE
- **Descrição:** Criar `src/components/YoutubeUrlInput.tsx` seguindo o mesmo padrão de props e comportamento de `FileUpload.tsx`.

  **Interface de props** (idêntica à de `FileUpload` exceto `selectedTranscriptionModel` que não é necessária aqui):
  ```tsx
  interface YoutubeUrlInputProps {
    onTranscriptionStart: () => void;
    onTranscriptionDone: (text: string) => void;
    onTranscriptionError: (error: string) => void;
    onTextLoaded: (text: string) => void; // não usado, mas mantido para consistência de interface
    isLoading: boolean;
  }
  ```

  **Comportamento:**
  - Renderizar um campo `<input type="url">` para a URL e um botão "Fetch transcript".
  - Quando o botão for clicado (ou Enter pressionado no campo):
    1. Chamar `onTranscriptionStart()`.
    2. Invocar `invoke<string>("fetch_youtube_transcript", { url })`.
    3. Em caso de sucesso: chamar `onTranscriptionDone(text)`.
    4. Em caso de erro: chamar `onTranscriptionError(String(err))`.
  - O botão deve estar `disabled` quando `isLoading === true` ou quando o campo estiver vazio.
  - O campo e o botão devem estar `disabled` quando `isLoading === true`.
  - Exibir um spinner ou texto "Fetching…" no botão durante o loading (mesmo padrão visual que `FileUpload` usa com `opacity-50 cursor-not-allowed`).
  - Não exibir mensagem de erro dentro do componente — o erro é propagado via `onTranscriptionError` e exibido pelo `App.tsx` (já existente).

  **Estilo:** seguir as mesmas classes Tailwind usadas em `FileUpload.tsx` para manter consistência visual. O container pode ser um `div` com `flex flex-col gap-3` (mesma estrutura do bloco "Paste text" no `App.tsx`). O campo deve usar as mesmas classes do `<textarea>` do modo paste: `bg-[#100c24] border border-[#2d2650] text-white rounded-lg px-4 py-3 text-sm`. O botão deve usar `bg-violet-600 hover:bg-violet-700`.
