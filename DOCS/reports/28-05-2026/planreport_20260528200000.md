## Decisões de design registradas

### D1 — `fetch_available_languages` mantida via `reqwest`

A função `fetch_available_languages` usa `reqwest` para buscar a watch page e extrair os idiomas de legenda disponíveis. Essa requisição **não requer autenticação** (o YouTube retorna `ytInitialPlayerResponse` com a lista de legendas mesmo sem sessão). Mantê-la via `reqwest` é mais simples e evita uma segunda navegação WebView2 só para isso. Nenhuma alteração nessa função.

### D2 — `fetch_timedtext` substituída, não extraída

A função `fetch_timedtext` (privada) será **removida** — sua lógica de tentativas múltiplas (json3/srv3 + kind=asr) será incorporada diretamente em `fetch_youtube_transcript` como lista de URLs candidatas passadas para o `eval_with_callback`. O parser `parse_caption_body` continua inalterado.

### D3 — `ytConnected` propagado via prop, não via `invoke` no componente filho

`YoutubeUrlInput` receberá `ytConnected: boolean` como prop vindo do `App.tsx`. O `App.tsx` inicializa esse estado via `invoke("youtube_is_connected")` no mount. Isso evita duplicação de chamadas ao backend e mantém o estado centralizado no `App`.

### D4 — Capability separada para janelas YouTube

Criar `src-tauri/capabilities/youtube.json` separado (em vez de modificar `default.json`) com `"windows": ["yt-login", "yt-fetch"]` e as permissões core necessárias. Isso mantém `default.json` limpo e com escopo apenas da janela `main`.

### D5 — Detecção de janela fechada no `youtube_login`

Para detectar que o usuário fechou a janela `yt-login` sem concluir o login, o polling usa `app.get_webview_window("yt-login").is_none()` em cada iteração. Se a janela não existir mais → retorna `false`.

### D6 — `youtube_disconnect` usando `clear_all_browsing_data`

O command `youtube_disconnect` chamará `win.clear_all_browsing_data()` na janela `yt-fetch` (se existir), depois fecha a janela. Se a janela não existir, cria uma temporária só para limpar e fecha em seguida. Isso limpa o cookie store do perfil WebView2 para `youtube.com`.

### Etapa 4 — Implementar `youtube_login`

- **Status:** PENDENTE
- **Descrição:** Adicionar command público async:
  ```rust
  #[tauri::command]
  pub async fn youtube_login(app: AppHandle) -> Result<bool, String>
  ```
  Implementação:
  1. Verificar se já existe uma janela `yt-login` aberta (`app.get_webview_window("yt-login")`); se sim, colocá-la em foco com `win.set_focus()` e retornar imediatamente sem criar outra.
  2. Criar janela de login:
     ```rust
     let win = WebviewWindowBuilder::new(&app, "yt-login",
         WebviewUrl::External(YT_LOGIN_URL.parse().map_err(|_| "url")?))
         .title("Conectar conta do YouTube")
         .inner_size(520.0, 720.0)
         .build()
         .map_err(|e| e.to_string())?;
     ```
  3. Loop de polling a cada 800ms:
     ```rust
     loop {
         tokio::time::sleep(Duration::from_millis(800)).await;
         // Detecta janela fechada pelo usuário
         if app.get_webview_window("yt-login").is_none() {
             return Ok(false);
         }
         let yt_url = YT_HOME.parse().unwrap();
         if let Ok(cookies) = win.cookies_for_url(yt_url) {
             if cookies.iter().any(|c| matches!(c.name(), "SAPISID" | "__Secure-3PSID")) {
                 let _ = win.close();
                 return Ok(true);
             }
         }
     }
     ```

### Etapa 6 — Refatorar `fetch_youtube_transcript`

- **Status:** PENDENTE

  Implementação:
  1. `let video_id = extract_video_id(&url)?;` — sem alteração.
  2. Criar client reqwest e chamar `fetch_available_languages` — sem alteração (ver D1).
  3. Para cada idioma da lista, construir as URLs candidatas (mesmo padrão atual: json3, json3+asr, srv3, srv3+asr):
     ```rust
     let candidates = [
         format!("https://www.youtube.com/api/timedtext?v={}&lang={}&fmt=json3", video_id, lang),
         format!("https://www.youtube.com/api/timedtext?v={}&lang={}&fmt=json3&kind=asr", video_id, lang),
         format!("https://www.youtube.com/api/timedtext?v={}&lang={}&fmt=srv3", video_id, lang),
         format!("https://www.youtube.com/api/timedtext?v={}&lang={}&fmt=srv3&kind=asr", video_id, lang),
     ];
     ```
  4. Para cada URL candidata, chamar `ensure_yt_webview(&app).await?` e executar via `eval_with_callback`:
     ```rust
     let js = format!(r#"
         (async () => {{
             try {{
                 const r = await fetch({url:?}, {{ credentials: "include" }});
                 return await r.text();
             }} catch (e) {{ return ""; }}
         }})()
     "#, url = timedtext_url);
     let (tx, rx) = oneshot::channel::<String>();
     let tx = std::sync::Mutex::new(Some(tx));
     win.eval_with_callback(js, move |json_val| {
         if let Some(sender) = tx.lock().unwrap().take() {
             let _ = sender.send(json_val);
         }
     }).map_err(|e| e.to_string())?;
     let raw = tokio::time::timeout(Duration::from_secs(15), rx)
         .await.map_err(|_| "timeout".to_string())?
         .map_err(|_| "canal fechado".to_string())?;
     // raw é uma JSON string contendo a string de resposta — desserializar 1 nível
     let body: String = serde_json::from_str(&raw).unwrap_or(raw);
     if let Some(text) = parse_caption_body(&body) {
         if !text.trim().is_empty() {
             return Ok(text);
         }
     }
     ```
  5. Se nenhum idioma/candidato retornar texto: `Err("Este vídeo não possui legendas disponíveis ou não foi possível acessá-las.".to_string())`.
  6. **Remover** a função `fetch_timedtext` (ela deixa de ser necessária — ver D2).
  7. Manter sem alteração: `extract_video_id`, `validate_video_id`, `extract_player_response_json`, `parse_caption_body`, `extract_text_from_xml`, `fetch_available_languages`, `DESKTOP_USER_AGENT`.
