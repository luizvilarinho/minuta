## 2. Respostas Г s perguntas de investigaГ§ГЈo

### 2.1 Abrir janela WebView2 para login

- API correta no Tauri 2.x: **`tauri::webview::WebviewWindowBuilder`** (nГЈo mais `WindowBuilder` da v1).
- Para abrir URL externa: `WebviewUrl::External(url)`.
  ```rust

  let login_url = "https://accounts.google.com/ServiceLogin?service=youtube".parse().unwrap();
  let win = WebviewWindowBuilder::new(&app, "yt-login", WebviewUrl::External(login_url))
      .title("Conectar conta do YouTube")
      .inner_size(520.0, 700.0)
      .on_navigation(move |url| { /* ver 2.2 */ true })
      .build()?;
  ```
- Fechar programaticamente: `win.close()?` (ou `app.get_webview_window("yt-login").unwrap().close()`).

### 2.2 Detectar que o login foi concluГ­do

Duas formas, ambas viГЎveis. **Recomendo a combinaГ§ГЈo das duas:**

1. **`on_navigation(|url: &Url| -> bool)`** вЂ” dispara em toda navegaГ§ГЈo top-level. DГЎ pra observar
   quando a URL muda para `https://www.youtube.com/` ou `.../feed/...` depois do login do Google.
   Гљtil como gatilho, mas a URL sozinha nГЈo garante sessГЈo.
2. **VerificaГ§ГЈo de cookie (fonte da verdade):** apГіs a navegaГ§ГЈo para `youtube.com`, ler os cookies
   e checar a presenГ§a dos cookies de sessГЈo (`SAPISID`, `SID`, `__Secure-3PSID`, `LOGIN_INFO`).
   Se existirem, o login estГЎ vГЎlido вЂ” fecha a janela e marca "conectado".

A API `WebviewWindow` no Tauri 2.11 expГµe:
```rust
pub fn cookies(&self) -> Result<Vec<Cookie<'static>>>
pub fn cookies_for_url(&self, url: Url) -> Result<Vec<Cookie<'static>>>
```
вљ пёЏ **Importante (Windows):** `cookies()`/`cookies_for_url()` **deadlocka se chamado dentro de um
command sГ­ncrono ou de um event handler**. Tem que ser chamado em **command `async`** (ou
`tauri::async_runtime::spawn` / thread separada). Isso vale tambГ©m para a leitura dentro do
`on_navigation` вЂ” nГЈo chame `cookies()` direto no callback; emita um evento/seta uma flag e leia
os cookies de um command async.

### 2.3 Compartilhamento de cookies entre janelas

**Sim, compartilham вЂ” desde que usem o mesmo perfil/`data_directory` do WebView2.**

- No Windows o WebView2 guarda cookies/localStorage num *user data folder* (perfil). Janelas do mesmo
  app que nГЈo definem `data_directory` usam o perfil padrГЈo do app в†’ **compartilham o cookie store
  automaticamente.** Logo, basta NГѓO setar `data_directory` (nem `incognito(true)`) na janela de login
  para que os cookies fiquem visГ­veis para o resto do app.
- `WebviewWindowBuilder` oferece `.data_directory(PathBuf)` e `.incognito(bool)` вЂ” para esta task,
  **nГЈo usar nenhum dos dois** (ou setar `incognito(false)`), garantindo perfil compartilhado.
- PersistГЄncia entre sessГµes do app: o perfil Г© gravado em disco, entГЈo os cookies **persistem apГіs
  fechar o app** вЂ” exatamente o que a task pede ("conectado/desconectado persiste entre sessГµes").
  NГЈo precisa salvar nada manualmente no `tauri-plugin-store` alГ©m de, opcionalmente, um flag de UI.
- Detalhe Windows a observar: a doc do Tauri avisa que migraГ§ГЈo para o esquema `http://tauri.localhost`
  pode resetar cookies do **app**; isso afeta os cookies *do prГіprio frontend*, **nГЈo** os de
  `youtube.com` (origem externa). NГЈo Г© problema para esta task, mas se aparecer reset inesperado,
  a saГ­da Г© `WebviewWindowBuilder::use_https_scheme(true)`.

### 2.4 Fetch autenticado via WebView2 вЂ” qual sub-abordagem

**Recomendada: NENHUMA das duas sub-abordagens da task. Use a abordagem C abaixo.**

- **Sub-abordagem A (injetar `fetch()` por JS e devolver ao Rust): VIГЃVEL e Г© a base da recomendaГ§ГЈo,
  com uma correГ§ГЈo.** NГЈo precisa do hack de IPC manual que circula na internet ("eval Г©
  fire-and-forget"). O Tauri **2.11 jГЎ tem retorno de valor nativo**:
  ```rust
  pub fn eval_with_callback(
      &self,
      js: impl Into<String>,
      callback: impl Fn(String) + Send + 'static,
  ) -> Result<()>
  ```
  O `callback` recebe o resultado serializado em JSON string. (A limitaГ§ГЈo "sem retorno" sГі vale para
  o `eval()` antigo.) PorГ©m hГЎ um ponto **crГ­tico de origem**: o `fetch()` sГі leva os cookies de
  sessГЈo e passa na checagem de CORS/SameSite do YouTube **se for executado a partir de uma pГЎgina
  com origem `youtube.com`**. Se a janela carregar nossa prГіpria pГЎgina local e fizer fetch
  cross-origin para `youtube.com`, o request vira cross-site e os cookies `SameSite` podem nГЈo ir.

- **Sub-abordagem B (navegar o webview direto para a URL do timedtext e capturar a resposta):
  NГѓO recomendada.** O Tauri tem `on_navigation` (cancela/observa navegaГ§ГЈo) e `on_page_load`, mas
  **nГЈo hГЎ hook para interceptar o corpo da resposta HTTP**. Ler o body exigiria parsear o DOM
  (`document.body.innerText`) depois do load вЂ” frГЎgil e sujo. Descartar.

- **Sub-abordagem C (RECOMENDADA) вЂ” janela oculta na origem youtube.com + `fetch` same-origin +
  `eval_with_callback`:**
  1. Abrir uma `WebviewWindow` **oculta** (`.visible(false)`) navegando para
     `https://www.youtube.com/` (mesma origem do timedtext).
  2. No `on_page_load` (evento `Finished`), rodar via `eval_with_callback` um `fetch()` *same-origin*
     para a URL do timedtext. Como a pГЎgina Г© youtube.com, o fetch Г© same-origin e **carrega os
     cookies de sessГЈo automaticamente**.
  3. O `callback` recebe o corpo da legenda (JSON3/srv3) jГЎ como string e devolve ao Rust.
  Isso resolve o problema de origem da sub-abordagem A e reaproveita todo o parser que jГЎ existe em
  `youtube.rs` (`parse_caption_body`).

### 2.5 DetecГ§ГЈo logado/deslogado sem abrir janela

- PossГ­vel **sem abrir janela visГ­vel**, mas o WebView2 precisa de um webview vivo para expor o cookie
  store. Na prГЎtica: ter (ou criar sob demanda, oculto) uma `WebviewWindow` apontando para
  `youtube.com` e chamar `cookies_for_url("https://www.youtube.com".parse()?)` num command async,
  procurando `SAPISID`/`__Secure-3PSID`. Se presentes в†’ logado.
- Como os cookies persistem no perfil em disco, esse check funciona em sessГµes futuras sem novo login.
- NГЈo hГЎ necessidade de plugin para ler cookies вЂ” `cookies()`/`cookies_for_url()` jГЎ fazem isso na
  versГЈo instalada.

### 2.6 Alternativas (plugins OAuth / http)

- **`tauri-plugin-oauth` / `tauri-plugin-localhost`:** servem para fluxos OAuth com *redirect* a um
  servidor local. O YouTube **nГЈo expГµe** OAuth pГєblico que dГЄ acesso ao `timedtext` de terceiros, e
  a YouTube Data API v3 (`captions.download`) sГі baixa legendas de vГ­deos **do prГіprio dono do canal**.
  Portanto **nГЈo atendem** o caso de uso (legendas de qualquer vГ­deo). Descartar.
- **`tauri-plugin-http`:** Г© sГі um `fetch` no frontend; nГЈo compartilha automaticamente os cookies de
  sessГЈo do perfil de login do YouTube de forma confiГЎvel e ainda cai no mesmo problema de origem.
  NГЈo traz vantagem sobre a abordagem C. Descartar.
- **ConclusГЈo:** nenhum plugin novo Г© necessГЎrio. Tudo com a API core do Tauri 2.11.

## 4. Riscos / pontos de atenГ§ГЈo

1. **Origem do fetch (crГ­tico):** o `fetch` do timedtext PRECISA rodar de uma pГЎgina `youtube.com`
   (abordagem C). Fetch cross-origin de uma pГЎgina local provavelmente falha por SameSite/CORS.
2. **Deadlock de cookies no Windows:** `cookies()`/`cookies_for_url()` **sГі** em command async / thread
   separada. Nunca dentro de `on_navigation`/event handler sГ­ncrono.
3. **`eval_with_callback` e tamanho:** legendas longas podem gerar strings grandes; o callback recebe
   JSON string. Em geral funciona, mas validar com vГ­deo longo. Se houver corte, encodar em base64
   no JS antes de devolver.
4. **Timing do login:** detectar conclusГЈo por cookie (nГЈo sГі por URL) evita falso positivo quando o
   usuГЎrio ainda estГЎ na tela de senha/2FA.
5. **Fragilidade contra mudanГ§as do YouTube:** mesma fragilidade que o cГіdigo atual jГЎ tem (parsing de
   `ytInitialPlayerResponse` / endpoint timedtext). A task nГЈo piora isso.
6. **Janela oculta no Windows:** WebView2 oculto (`visible(false)`) funciona para executar JS, mas o
   `on_page_load` precisa disparar вЂ” confirmar que o load completa em janela nГЈo-visГ­vel (caso raro de
   nГЈo disparar, mostrar a janela 1x no login resolve, e nas buscas seguintes os cookies jГЎ existem).

## 6. RecomendaГ§ГЈo clara de implementaГ§ГЈo

Implementar a **abordagem C**: janela oculta em `youtube.com` + `fetch` same-origin via
`eval_with_callback`, com perfil WebView2 compartilhado (sem `data_directory`/`incognito`), e detecГ§ГЈo
de login por cookie (`SAPISID`/`__Secure-3PSID`) em commands **async**.

### Esqueleto Rust (orientativo)

```rust
use tauri::{AppHandle, WebviewUrl, WebviewWindowBuilder, Manager};
use tauri::webview::PageLoadEvent;
use tokio::sync::oneshot;

const YT_HOME: &str = "https://www.youtube.com/";

// --- 1. DetecГ§ГЈo de login (async obrigatГіrio no Windows) ---
#[tauri::command]
async fn youtube_is_connected(app: AppHandle) -> Result<bool, String> {
    // garante um webview vivo apontando para youtube (oculto) e lГЄ cookies
    let win = ensure_yt_webview(&app).await?;
    let url = YT_HOME.parse().map_err(|_| "url".to_string())?;
    let cookies = win.cookies_for_url(url).map_err(|e| e.to_string())?;
    Ok(cookies.iter().any(|c| matches!(c.name(), "SAPISID" | "__Secure-3PSID" | "SID")))
}

// --- 2. Login: abre janela VISГЌVEL, fecha quando cookie aparece ---
#[tauri::command]
async fn youtube_login(app: AppHandle) -> Result<bool, String> {
    let url = "https://accounts.google.com/ServiceLogin?service=youtube"
        .parse().map_err(|_| "url".to_string())?;
    let win = WebviewWindowBuilder::new(&app, "yt-login", WebviewUrl::External(url))
        .title("Conectar conta do YouTube")
        .inner_size(520.0, 720.0)
        .build().map_err(|e| e.to_string())?;

    // polling simples num loop async atГ© achar o cookie de sessГЈo (ou janela fechar)
    loop {
        tokio::time::sleep(std::time::Duration::from_millis(800)).await;
        if win.is_closable().is_err() { return Ok(false); } // janela fechada pelo usuГЎrio
        let yt = YT_HOME.parse().unwrap();
        if let Ok(cookies) = win.cookies_for_url(yt) {
            if cookies.iter().any(|c| matches!(c.name(), "SAPISID" | "__Secure-3PSID")) {
                let _ = win.close();
                return Ok(true);
            }
        }
    }
}

// --- 3. Fetch autenticado da legenda (abordagem C) ---
#[tauri::command]
async fn fetch_youtube_transcript(app: AppHandle, url: String) -> Result<String, String> {
    let video_id = extract_video_id(&url)?;            // jГЎ existe
    let win = ensure_yt_webview(&app).await?;          // janela oculta em youtube.com

    // descobre idiomas como hoje (pode ser via fetch same-origin do watch page tambГ©m)
    let timedtext = format!(
        "https://www.youtube.com/api/timedtext?v={}&lang=pt&fmt=json3", video_id
    );

    let js = format!(r#"
        (async () => {{
            try {{
                const r = await fetch("{timedtext}", {{ credentials: "include" }});
                return await r.text();
            }} catch (e) {{ return ""; }}
        }})()
    "#);

    let (tx, rx) = oneshot::channel::<String>();
    let tx = std::sync::Mutex::new(Some(tx));
    win.eval_with_callback(js, move |json| {
        if let Some(tx) = tx.lock().unwrap().take() {
            let _ = tx.send(json); // 'json' = string JSON serializada do retorno
        }
    }).map_err(|e| e.to_string())?;

    let raw = tokio::time::timeout(std::time::Duration::from_secs(15), rx)
        .await.map_err(|_| "timeout".to_string())?
        .map_err(|_| "canal".to_string())?;

    // 'raw' Г© uma JSON string contendo a string da legenda -> desserializa 1 nГ­vel
    let body: String = serde_json::from_str(&raw).unwrap_or(raw);
    parse_caption_body(&body)                          // parser jГЎ existe
        .ok_or_else(|| "Sem legendas acessГ­veis.".to_string())
}
```

> os cookies de sessГЈo vГЈo junto.
