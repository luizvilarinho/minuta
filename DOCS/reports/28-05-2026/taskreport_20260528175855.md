# Task: Analisar falha no fetch de legendas do YouTube via WebView2

## Objective
Investigar em profundidade por que o fluxo atual de YouTube falha ao buscar legendas/transcrição, mesmo conseguindo carregar a watch page, retornando ao usuário a mensagem "Este vídeo não possui legendas disponíveis ou não foi possível acessá-las." O objetivo é identificar a causa mais provável do problema e propor a direção de correção mais concreta possível.

## Context
O app desktop Windows em Tauri + React + Rust já possui uma implementação em `src-tauri/src/commands/youtube.rs` baseada em WebView2/cookies de sessão para acessar páginas do YouTube e tentar buscar legendas via `api/timedtext`. Os logs atuais mostram que a watch page HTML é carregada com sucesso, mas todas as tentativas de fetch para URLs de legenda retornam body vazio (`result len: 0`), inclusive quando a URL vem assinada do próprio `ytInitialPlayerResponse`.

## Requirements
- Analisar o código atual do fluxo YouTube e os logs fornecidos.
- Identificar hipóteses técnicas para o retorno vazio nas requisições de `timedtext`.
- Verificar se o problema está mais ligado a autenticação/cookies, método de fetch via WebView, formato/URL das legendas, headers/origin/referer, limitações do XHR síncrono, capabilities do Tauri/WebView2, ou comportamento do YouTube.
- Comparar hipóteses com o que os logs efetivamente mostram.
- Entregar uma conclusão priorizada com a causa mais provável e próximos passos recomendados.

## Constraints
- Nesta etapa, fazer análise profunda; não implementar código ainda.
- Considerar apenas o codebase atual e os logs fornecidos, sem assumir sucesso de soluções externas não testadas.
- Separar sintomas observados de inferências.
- Se a análise indicar que a arquitetura atual está bloqueada, deixar isso explícito.

## Edge Cases
- Watch page carrega, mas timedtext retorna corpo vazio para URL assinada e não assinada.
- Sessão parece existir, mas não é efetivamente aplicada nas requests de legenda.
- O WebView consegue navegar em `youtube.com/watch`, mas o fetch/XHR para `youtube.com/api/timedtext` sofre restrição de cookies, CORS, same-site, permissions policy ou comportamento específico do WebView2.
- O signed `baseUrl` pode exigir parâmetros adicionais ou outro modo de acesso além do fetch atual.
- O problema pode estar no mecanismo de transporte (`eval_with_callback` + XHR síncrono) e não nas legendas em si.
