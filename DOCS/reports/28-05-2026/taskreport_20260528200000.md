# Task: Login YouTube para autenticação de legendas

## Objective
Permitir que o usuário conecte sua conta do YouTube ao app, de forma que o app possa buscar transcrições de vídeos usando a sessão autenticada. O login é necessário porque o YouTube requer cookies de sessão válidos para servir legendas via API.

## Context
A funcionalidade de resumo de YouTube já existe (youtube.rs + YoutubeUrlInput.tsx), mas falha ao buscar legendas porque o timedtext API retorna body vazio sem autenticação. A solução é usar o WebView2 nativo do Tauri (já presente no app por ser desktop Windows) para:
1. Fazer o usuário logar no YouTube uma vez
2. Usar essa sessão autenticada para buscar legendas em requests subsequentes

## Requirements
- Botão "Conectar conta do YouTube" na tela de Settings
- Ao clicar, abre uma janela WebView2 com youtube.com para o usuário fazer login
- Após login confirmado, Settings exibe "YouTube conectado" com opção de desconectar
- Ao buscar transcrição, o app detecta se está logado; se não, orienta o usuário a conectar
- O fetch da transcrição usa o WebView2 (com cookies da sessão) em vez de reqwest puro
- O estado "conectado/desconectado" persiste entre sessões do app

## Constraints
- Não armazenar credenciais — usar apenas os cookies da sessão do WebView2
- O WebView2 do Tauri compartilha cookie store entre janelas do mesmo app — aproveitar isso
- Compatível com o backend Rust/Tauri 2.0 existente
- Não modificar o fluxo de resumo (generate_summary) nem outros modos de input

## Edge Cases
- Usuário tenta buscar transcrição sem estar logado → mensagem clara pedindo para conectar
- Sessão do YouTube expirou → erro tratado, orientar a reconectar
- Usuário fecha a janela de login sem concluir → estado permanece "desconectado"
- Vídeo privado ou sem legendas mesmo com login → mensagens de erro existentes já cobrem
