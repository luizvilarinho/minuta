# Task Report — Redesign de Layout

**Data:** 28-05-2026  
**Pipeline:** Encurtado (Task Report → Designer → Avaliação direta)  
**Solicitante:** Coordenador  

---

## Objetivo

Redesenhar o layout do app Volp de 3 colunas independentes para um layout de 2 zonas mais claro e com navegação por abas.

---

## Problema atual

- Col 1 empilha ModelSelector, FileUpload, Paste Text e Record sem hierarquia — o usuário não sabe o que deve fazer primeiro.
- O ModelSelector de transcrição aparece para todos os modos, mas só é necessário no modo Record e Upload de vídeo (mp4).
- Col 2 mistura a transcrição editável com todos os controles de geração de resumo, sem separação visual clara.
- Col 3 só exibe o resumo, desperdiçando espaço em telas maiores.

---

## Escopo das mudanças

### Coluna 1 — Input (sidebar fixa, `280px`)

Substituir o empilhamento atual por um seletor de modo com submenu (tabs ou radio buttons estilizados):

| Modo | O que mostrar |
|---|---|
| **Upload File** | Componente `FileUpload` (MP4, TXT, MD) |
| **Paste Text** | Textarea de colagem de texto (o que hoje está no collapse "Paste text") |
| **Record** | Botão Record/Stop + cronômetro + `ModelSelector` de transcrição + botão Retry se houver erro |

- O `ModelSelector` de transcrição **só aparece no modo Record** (e internamente o FileUpload já lida com o modelo ao receber um MP4, passado via prop).
- Erros de transcrição (`transcriptionError`) e o botão Resume (`canResume`) permanecem visíveis abaixo do conteúdo ativo.
- O erro genérico de API (`error && !noKeyMessage`) permanece no topo da coluna.

### Coluna 2 — Conteúdo (ocupa o restante, com abas)

Duas abas no topo: **Transcription** e **Summary**.

**Aba Transcription:**
- Textarea editável com altura generosa (`min-h-[400px]` ou `flex-1`).
- Contador de caracteres abaixo.
- Loading overlay quando `isTranscribing`.

**Aba Summary (2 sub-colunas):**
- Sub-coluna esquerda (controles, ~`380px` fixo ou `40%`):
  - `ModelSelector` (summary model)
  - `SummaryControls` completo (system prompt, prompt adicional com library, toggle soma/substitui, botão Generate Summary)
- Sub-coluna direita (leitura):
  - `SummaryReadView` com mensagem de estado vazio: *"Click Generate Summary to see the summary here."*
  - `SaveButton` abaixo do resumo

## Arquivos a modificar

| Arquivo | Mudança |
|---|---|
| `src/App.tsx` | Restructurar layout: col 1 com abas de input, col 2 com abas Transcription/Summary |

| `src/components/SummaryReadView.tsx` | Ajustar mensagem do estado vazio para *"Click Generate Summary to see the summary here."* |

Os componentes `FileUpload`, `ModelSelector`, `SummaryControls`, `SaveButton` **não precisam de mudança interna** — apenas reposicionamento.

---

## Restrições

- Manter paleta de cores e tokens visuais existentes (`#0e0a1a`, `#1f1a36`, `#2d2650`, violet-400/500/600, etc.).
- Não alterar nenhuma lógica de negócio, hooks ou chamadas Tauri.
- Layout responsivo: abaixo de `xl` as colunas empilham (comportamento atual preservado).
- Não criar novos componentes além do necessário — priorizar mudanças no `App.tsx`.

- Usuário consegue identificar claramente qual modo de input usar.
- ModelSelector de transcrição só aparece no modo Record.
- Aba Summary mostra controles e resumo lado a lado.
- SaveButton está abaixo do resumo na aba Summary.
- App compila sem erros TypeScript.
