# Task: Web Page Summarization + Input Column Resize

## Objective
Adicionar um novo modo "Web" ao app que permite ao usuário colar uma URL, extrair o texto da página no backend Rust e exibi-lo no campo de transcrição — de onde o fluxo normal de resumo segue. Em paralelo, aumentar a largura da coluna "Input" para acomodar o 5º botão de modo sem aperto visual.

## Context
- `src/App.tsx` — layout principal, seletor de modo (`inputMode`), grid da coluna Input
- `src/components/YoutubeUrlInput.tsx` — componente de referência para o novo `WebUrlInput.tsx`
- `src-tauri/src/commands/youtube.rs` — padrão arquitetural de referência para o novo comando Rust
- `src-tauri/src/commands/mod.rs` e `src-tauri/src/lib.rs` — onde o novo comando deve ser registrado
- `src-tauri/Cargo.toml` — onde a nova dependência de parsing HTML deve ser adicionada

## Requirements
- Novo modo `"webpage"` adicionado ao `inputMode` em `App.tsx`
- Novo componente `WebUrlInput.tsx` com campo de URL, botão de busca e estados de loading/erro
- Novo Tauri command `fetch_webpage_text(url: String) -> Result<String, String>` no backend Rust
  - Faz HTTP GET na URL com `reqwest` (já no projeto)
  - Extrai o texto legível do HTML com a crate `html2text` (a adicionar no `Cargo.toml`)
  - Retorna o texto bruto para o frontend
- O texto extraído popula o campo de transcrição exatamente como os outros modos fazem
- O usuário decide se quer prosseguir para o resumo ou não — sem lógica automática de continuação
- Modelo de transcrição **não é exibido** neste modo (não há transcrição de áudio)
- Coluna "Input" aumentada de `340px` para `380px` (ou valor suficiente para 5 botões sem aperto)
- Seletor de modo atualizado de `grid-cols-4` para `grid-cols-5`

## Constraints
- A chave da API nunca passa pelo frontend — mas neste caso não há chamada autenticada; o GET na URL é público, portanto pode ser feito no backend Rust sem envolver a chave
- Manter o padrão visual dos outros modos (labels, espaçamento, estados de erro)
- Não quebrar nenhum dos 4 modos existentes
- Não adicionar lógica de headless browser ou execução de JavaScript — apenas GET + parse de HTML estático

## Edge Cases
- URL inválida (não começa com http/https) → erro claro antes de fazer qualquer request
- Página retorna HTML quase vazio (SPA com JS) → o texto extraído será mínimo; o usuário vê isso no campo de transcrição e decide não resumir — comportamento esperado, sem tratamento especial
- Timeout / erro de rede → exibir mensagem de erro no componente
- Página retorna conteúdo muito longo → retornar como está; o modelo de resumo lida com isso
