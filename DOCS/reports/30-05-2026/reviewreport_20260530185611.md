## 1. `src-tauri/src/commands/webpage.rs`

### O que foi feito
Novo comando Tauri `fetch_webpage_text(url: String) -> Result<String, String>` que:
1. Valida se a URL começa com `http://` ou `https://`
2. Cria um `reqwest::Client` com timeout de 15 s
3. Faz GET com `User-Agent: Mozilla/5.0`
4. Lê o corpo da resposta como texto
5. Converte HTML para texto legível via `html2text::from_read` dentro de `spawn_blocking`

### Achados

**[MÉDIO] Sem verificação do status HTTP**

O código não checa `response.status()` antes de ler o body. Uma resposta 404 ou 500 será silenciosamente convertida para texto (conteúdo da página de erro) e retornada ao frontend como se fosse sucesso. O usuário verá o HTML do erro no campo de transcrição sem saber que a página não existia.

Sugestão:
```rust
if !response.status().is_success() {
    return Err(format!(
        "A página retornou o status HTTP {}",
        response.status()
    ));
}
```

**[BAIXO] `User-Agent` muito genérico**

`"Mozilla/5.0"` é um fragmento inválido — a maioria dos CDNs e WAFs considera isso suspeito. Usar a string completa comum (ex.: `"Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"`) aumenta a taxa de sucesso em páginas com bot-detection leve. Não é crítico para o MVP, mas pode gerar erros difíceis de diagnosticar.

**[BAIXO] `html2text::from_read` retorna `Result` com erro genérico**

O tipo de erro de `html2text 0.17.x` pode carregar informação útil para diagnóstico. A conversão atual via `map_err` preserva a mensagem, então o tratamento está correto — não há problema, apenas uma observação.

**[OK] `spawn_blocking` para operação síncrona**
Correto e necessário: `html2text::from_read` é síncrono e pode bloquear o executor Tokio em páginas grandes. O uso de `spawn_blocking` está alinhado com o padrão do projeto.

**[OK] Timeout configurado**
15 segundos de timeout é adequado e consistente com os outros comandos do projeto.

**[OK] Validação de prefixo http/https**
Cobre o caso de edge especificado na task (URL sem protocolo).

## 6. `src/App.tsx`

### Achados

**[CRÍTICO] `setContentTab("transcription")` em vez de `"summary"` no modo webpage**

No handler `onTranscriptionDone` do `WebUrlInput`, o código faz:
```tsx
setContentTab("transcription")
```
Todos os outros modos (upload, record, youtube) fazem `setContentTab("summary")` ao concluir, levando o usuário diretamente para a aba de resumo. O modo webpage é o único que mantém o usuário na aba de transcrição após extrair o texto.

O comportamento intencional descrito na task é:
> "O usuário decide se quer prosseguir para o resumo ou não — sem lógica automática de continuação"

Isso gera ambiguidade de interpretação: a task diz "sem lógica automática", mas todos os outros modos fazem a navegação automática para a aba "Summary". Se a intenção era manter consistência com os demais modos (navegar para Summary automaticamente), o valor deveria ser `"summary"`. Se a intenção era deixar o usuário na aba de transcrição para revisar o conteúdo extraído antes de resumir, o valor `"transcription"` é correto mas diverge do padrão dos outros modos.

**Recomendação:** Alinhar com o product owner. A leitura mais natural da frase "sem lógica automática de continuação" é que não deve haver nenhuma ação automatizada além de mostrar o texto — navegar para "summary" automaticamente seria considerado "continuação automática". Portanto `"transcription"` pode ser a escolha correta, mas deve ser documentada como decisão explícita.

**[OK] `inputMode` estendido com `"webpage"`**
```tsx
const [inputMode, setInputMode] = useState<"upload" | "paste" | "record" | "youtube" | "webpage">("upload");
```
Tipo union atualizado corretamente.

**[OK] `ModelSelector` não aparece no modo `"webpage"`**
A condição de exibição é:

```tsx
{(inputMode === "upload" || inputMode === "record" || inputMode === "youtube") && (
  <ModelSelector ... />

```
`"webpage"` e `"paste"` estão corretamente excluídos.

**[OK] Seletor de 5 botões**

Atualizado de `grid-cols-4` para `grid-cols-5`. Array de modos inclui o novo `{ key: "webpage", label: "Web" }`.

**[OK] Coluna Input alargada**
```tsx
<main className="flex-1 grid grid-cols-1 xl:grid-cols-[380px_1fr]">
```

Largura aumentada de `340px` para `380px`.

**[OK] `setPastedText("")` chamado no `onTranscriptionDone` do webpage**
Correto — evita que texto colado anteriormente conflite com o texto extraído da web.

**[OK] `setCanResume(false)` chamado corretamente**
