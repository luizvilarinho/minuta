# Task: Biblioteca de prompts adicionais salvos

## Objective
Adicionar ao componente `SummaryControls` a capacidade de salvar prompts adicionais nomeados em uma biblioteca persistente. O usuário poderá ver a lista de prompts salvos, escolher o mais adequado para carregar no campo de prompt adicional, e criar/salvar novos prompts com nome.

## Context
- Componente afetado: `src/components/SummaryControls.tsx`
- O campo de prompt adicional já existe e é persistido via `localStorage` (chave `volp_additional_prompt`)
- A persistência da biblioteca de prompts também deve usar `localStorage`
- A interface atual tem: textarea de prompt adicional, toggle soma/substitui e botão gerar

## Requirements
- O usuário pode salvar o prompt atual com um nome, criando uma entrada na biblioteca
- O usuário pode ver a lista de prompts salvos e selecionar um para carregar no textarea
- O usuário pode criar um novo prompt diretamente (nome + texto) a partir da lista
- O usuário pode excluir prompts da biblioteca
- A biblioteca persiste via `localStorage` (chave `volp_saved_prompts`, formato JSON array: `[{ id, name, text }]`)
- Ao selecionar um prompt da lista, o textarea é preenchido com o texto desse prompt
- A UI deve seguir o estilo visual existente do app (fundo `#18142b`, bordas `#2d2650`, accent `violet`)

## Constraints
- Não alterar o fluxo de geração de resumo nem o toggle soma/substitui
- Não usar `tauri-plugin-store` - `localStorage` é suficiente e já é o padrão do projeto para esse tipo de dado
- Não criar novos arquivos de componente além do necessário - a lógica pode ficar dentro de `SummaryControls.tsx`
- Não alterar outros componentes além de `SummaryControls.tsx`
- Manter o comportamento existente: edição livre do textarea continua funcionando normalmente

## Edge Cases
- Biblioteca vazia: mostrar estado vazio com CTA para criar o primeiro prompt
- Nome duplicado: permitir (não é um erro), o usuário decide
- Prompt com texto vazio: não deve ser salvo
- Prompt sem nome: não deve ser salvo
- Ao carregar um prompt da lista, o textarea é sobrescrito com o texto do prompt selecionado
