## Decisão: react-markdown vs. formatação manual

**Decisão: Opção B — Formatação manual via split/regex.**

Justificativa:
- O DEFAULT_SYSTEM_PROMPT produz sempre o mesmo esquema de seções (`## Summary`, `## Key Points`, `## Action Items / Next Steps`, `## People Mentioned`, `## Open Questions`).
- Dividir por `\n\n`, detectar linhas que começam com `##` e renderizar como headings é trivial e cobre 100% dos casos de uso.
- Elimina dependência externa (`react-markdown` + `remark`), reduz bundle e risco de conflito de versão.
- O codeAgent não precisa de aprovação do Orquestrador para instalar pacotes — economiza um ciclo de handoff.

### Etapa 2 — Criar componente `SummaryReadView`

**Arquivo novo:** `src/components/SummaryReadView.tsx`

Componente de leitura do resumo, sem controles de edição. Renderiza o Markdown fixo com formatação manual.

**Props:**
```ts
interface SummaryReadViewProps {
  summary: string;
  isLoading: boolean; // loading de geração em curso
}
```

**Comportamento:**
- `summary` vazio + `isLoading` false → placeholder: mensagem neutra informando que o resumo aparecerá aqui após geração.
- `summary` vazio + `isLoading` true → spinner centralizado + texto "Generating summary…".
- `summary` preenchido + `isLoading` false → renderização do conteúdo.
- `summary` preenchido + `isLoading` true → exibe conteúdo anterior com overlay de loading sutil (indicando nova geração em curso).

**Formatação do Markdown fixo:**

Função utilitária `parseSummaryMarkdown(text: string)` que:
1. Split por `\n\n` para separar blocos.
2. Para cada bloco, se começa com `## `, renderiza como `<h2>` (heading da seção).
3. Caso contrário, renderiza como `<p>` ou lista `<ul>/<li>` se linhas começam com `- ` ou `* `.
4. Mantém quebras de linha simples dentro de blocos de lista.

**Tipografia (classes Tailwind):**
- Container: `overflow-y-auto` com `max-h-[calc(100vh-120px)]` (scroll interno sem crescer a página)
- Heading `## ...`: `text-violet-300 text-sm font-semibold uppercase tracking-wider mb-2 mt-5 first:mt-0`
- Parágrafos: `text-gray-200 text-sm leading-7`
- Lista: `list-disc list-inside text-gray-200 text-sm leading-7 space-y-1`
- Separador entre seções: linha `border-t border-[#2d2650]` opcional ou só margin-top
