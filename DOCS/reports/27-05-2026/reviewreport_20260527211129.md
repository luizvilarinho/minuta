## Categorias de Revisão

### 1. Correção

**[MENOR] `error?.includes("não configurada")` nunca será verdadeiro após a globalização**
- Arquivo: `src/App.tsx:138`
- O backend Rust ainda devolve a mensagem em português `"não configurada"`. A string de verificação foi mantida intacta na lógica do frontend, o que é tecnicamente correto para o estado atual. Porém, se o backend for traduzido futuramente, `noKeyMessage` nunca será gerado e o aviso amarelo no ModelSelector desaparecerá silenciosamente.
- Risco imediato: nenhum (backend não foi alterado, conforme constraint da tarefa).
- Recomendação: registrar como debt técnico ou adicionar comentário.

**[MENOR] Exclusão mútua incompleta entre Paste text e File/Recording**
- Arquivo: `src/App.tsx`
- A tarefa exige que "limpar um limpa o outro". Quando o usuário cola texto (`setPastedText + setTranscription`), o campo da transcrição é atualizado corretamente. Porém, ao carregar um arquivo (`onTextLoaded` / `onTranscriptionDone`), o `pastedText` state NÃO é zerado — apenas `transcription` é sobrescrito. Isso significa que ao fechar e reabrir o "Paste text" collapsible após um upload, o texto colado antigo ainda aparece, enquanto a transcrição exibida é a do arquivo.
- Impacto: inconsistência de estado visível para o usuário; não quebra funcionalidade principal.

**[MENOR] `promptMode` lido do localStorage sem validação de valor inválido**
- Arquivo: `src/components/SummaryView.tsx:37`
- O cast `as "sum" | "replace"` não valida se o valor no localStorage é de fato um dos dois. Se o usuário (ou outro código) gravar um valor diferente, o estado ficará inválido. O default `?? "sum"` cobre apenas `null`, não valores corrompidos.
- Sugestão: `const stored = localStorage.getItem("volp_prompt_mode"); return stored === "replace" ? "replace" : "sum";`
